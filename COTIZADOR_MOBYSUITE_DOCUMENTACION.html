📋 Documentación Cotizador Web Mobysuite v2.0.4

Proyecto: Boulevard Santa Rosa

Fecha: 2026-06-25

1. Implementación

El cotizador se integra en la página mediante un script que inyecta un contenedor.

Estructura Base HTML:

<!-- Contenedor donde se monta el cotizador --> <div id="mobysuite-cotizador"></div> <!-- Script de inyección --> <script src="https://[URL-MOBYSUITE]/cotizador.js" data-real-estate="Boulevard Santa Rosa" data-project-id="[PROJECT-ID]" data-container="mobysuite-cotizador" ></script>
✓ IMPORTANTE: El script se inyecta automáticamente. Se configura con atributos data-* en el script.

2. Parámetros de Configuración

Parámetros Obligatorios (NO se pueden cambiar):

Parámetro Tipo Descripción
real-estate Text Obligatorio. Nombre de la inmobiliaria como se encuentra registrada en Mobysuite
project-id Numérico Obligatorio. ID del proyecto en Mobysuite

Parámetros Customizables (Sí se pueden cambiar):

Parámetro Tipo Default Descripción
template Text "default" Plantilla a utilizar para el cotizador. Define la estructura visual base.
country-code Text "CL" Código del país (ISO 3166-1). Para Chile: "CL"
show-real-estate-logo Boolean true Mostrar logo de la inmobiliaria en el cotizador
real-estate-logo-url Text null URL del logo de la inmobiliaria. Si no se proporciona, usa el registrado en Mobysuite
show-project-logo Boolean true Mostrar logo del proyecto en el cotizador
primary-color Text (Hex) #0671AE Color primario del cotizador. Formato: #RRGGBB
secondary-color Text (Hex) #84CE25 Color secundario (acentos, botones). Formato: #RRGGBB
hide-selected-assets Boolean false Ocultar tabla de bienes con precios precisos. Si es true, muestra tabla sin precios específicos
use-secondary-image Boolean false Mostrar segunda imagen de cada bien registrada en Mobysuite
success-view-type Text "simple" Tipo de vista después de envío. Opciones: "simple", "detailed"
typology Text null Opcional. Filtrar por tipología específica (ej: "departamento", "bodega")

3. Campos del Formulario (NO Customizable)

⚠️ IMPORTANTE: Los siguientes campos son obligatorios y NO se pueden cambiar. Los trae Mobysuite automáticamente:
Campo Tipo Obligatorio Descripción

apellido Text ✓ Sí Apellido del interesado

6. Solución de Problemas

Error: "¡Que lastima! Ya no quedan más propiedades disponibles para cotizar"

Significa que no hay propiedades disponibles para el filtro actual. Verificar:

  • El project-id es correcto
  • Hay propiedades disponibles en Mobysuite para ese proyecto
  • El país/país-code es correcto

El formulario no valida correctamente

Asegurarse de que:

  • Los campos obligatorios estén presentes (nombre, apellido, correo, teléfono)
  • El email sea válido
  • El teléfono tenga formato válido para Chile
